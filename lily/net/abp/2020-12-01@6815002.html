# ABP 的数据过滤器

# 预定义的过滤器

ABP 预定义了一些过滤器

# ISoftDelete 软删除

实体类实现 ISoftDelete 接口，拥有软删除的功能，将实体标记为删除而不在数据库中实际的进行删除。

using System;
using Volo.Abp;
using Volo.Abp.Domain.Entities;

namespace Acme.BookStore
{
    public class Book : AggregateRoot<Guid>, ISoftDelete
    {
        public string Name { get; set; }

        public bool IsDeleted { get; set; }
    }
}
  • IsDeleted 当进行删除操作时，ABP 会自动将该属性的值设置为 true。

在仓储上调用 HardDeleteAsync 时会将实体真正删除。

# IMultiTenant 多租户

用于隔离租户与租户之间的数据，实现 IMultiTenant 接口可以使实体查询时拥有该功能。

using System;
using Volo.Abp;
using Volo.Abp.Domain.Entities;
using Volo.Abp.MultiTenancy;

namespace Acme.BookStore
{
    public class Book : AggregateRoot<Guid>, IMultiTenant
    {
        public string Name { get; set; }

        public Guid? TenantId { get; set; }
    }
}
  • TenantId 该属性用于自动过滤当前租户的实体数据。

# 启用/禁用数据过滤器

使用 IDataFilter 服务控制过滤器

using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using Volo.Abp;
using Volo.Abp.Data;
using Volo.Abp.DependencyInjection;
using Volo.Abp.Domain.Repositories;

namespace Acme.BookStore
{
    public class MyBookService : ITransientDependency
    {
        private readonly IDataFilter _dataFilter;
        private readonly IRepository<Book, Guid> _bookRepository;

        public MyBookService(IDataFilter dataFilter,IRepository<Book, Guid> bookRepository)
        {
            _dataFilter = dataFilter;
            _bookRepository = bookRepository;
        }

        public async Task<List<Book>> GetAllBooksIncludingDeletedAsync()
        {
            //暂时禁用ISoftDelete过滤器
            using (_dataFilter.Disable<ISoftDelete>())
            {
                return await _bookRepository.GetListAsync();
            }
        }
    }
}

除了禁用过滤器还有

  • IDataFilter.Enable() 启用过滤器
  • IDataFilter.IsEnabled() 检查是否启用了过滤器

这里为什么是"暂时禁用ISoftDelete过滤器"，因为ABP默认是自动开启数据过滤的。

尽量的都在 using 块内使用，以确保执行完之后将过滤器重置为以前的状态。

# 另一种用法

using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using Volo.Abp;
using Volo.Abp.Data;
using Volo.Abp.DependencyInjection;
using Volo.Abp.Domain.Repositories;

namespace Acme.BookStore
{
    public class MyBookService : ITransientDependency
    {
        private readonly IDataFilter<ISoftDelete> _softDeleteFilter;
        private readonly IRepository<Book, Guid> _bookRepository;

        // 在这里就指定了过滤的类型
        public MyBookService(IDataFilter<ISoftDelete> softDeleteFilter, IRepository<Book, Guid> bookRepository)
        {
            _softDeleteFilter = softDeleteFilter;
            _bookRepository = bookRepository;
        }

        public async Task<List<Book>> GetAllBooksIncludingDeletedAsync()
        {
            using (_softDeleteFilter.Disable())
            {
                return await _bookRepository.GetListAsync();
            }
        }
    }
}
  • 在注入时就进行过滤类型的筛选，这样就不用在使用时进行设定了。

# 过滤器的全局配置

用于设定过滤器的全局配置选项

Configure<AbpDataFilterOptions>(options =>
{
    options.DefaultStates[typeof(ISoftDelete)] = new DataFilterState(isEnabled: false);
});
  • 此设定为默认条件下禁用 ISoftDelete 过滤器。

# 自定义过滤器

首先就是为过滤器定义一个接口，然后在实体类实现它。

public interface IIsActive
{
    bool IsActive { get; }
}

public class Book : AggregateRoot<Guid>, IIsActive
{
    public string Name { get; set; }

    public bool IsActive { get; set; }
}

# EntityFramework Core

ABP将 EF Core 的全局查询过滤器系统用于 EF Core 集成。所以直接使用 DbContext 也可以按预期的情况执行。

使用自定义过滤器的最佳方法是为 DbContext 重写 ShouldFilterEntity 和 CreateFilterExpression 方法

protected bool IsActiveFilterEnabled => DataFilter?.IsEnabled<IIsActive>() ?? false;

protected override bool ShouldFilterEntity<TEntity>(IMutableEntityType entityType)
{
    if (typeof(IIsActive).IsAssignableFrom(typeof(TEntity)))
    {
        return true;
    }

    return base.ShouldFilterEntity<TEntity>(entityType);
}

protected override Expression<Func<TEntity, bool>> CreateFilterExpression<TEntity>()
{
    var expression = base.CreateFilterExpression<TEntity>();

    if (typeof(IIsActive).IsAssignableFrom(typeof(TEntity)))
    {
        Expression<Func<TEntity, bool>> isActiveFilter =
            e => !IsActiveFilterEnabled || EF.Property<bool>(e, "IsActive");
        expression = expression == null 
            ? isActiveFilter 
            : CombineExpressions(expression, isActiveFilter);
    }

    return expression;
}
  • IsActiveFilterEnabled 属性用于检查是否启用了 IIsActive 过滤。
  • 重写 ShouldFilterEntity 和 CreateFilterExpression 方法，用于检查实体是否实现了 IIsActive 接口，并在必要时组合查询表达式。
