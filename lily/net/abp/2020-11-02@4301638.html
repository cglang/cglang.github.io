# 安装 ABP CLI 以及常用命令

# 开始

# 安装 ABP CLI

ABP CLI是使用ABP框架启动新解决方案的最快方式. 使用命令行窗口安装ABP CLI:

dotnet tool install -g Volo.Abp.Cli

如果已经安装了 ABP CLI 可使用下面的命令更新:

dotnet tool update -g Volo.Abp.Cli

# 常用命令

中文文档 (opens new window) 英文文档 (opens new window)

在一个空文件夹使用 abp new 命令创建新解决方案:

# 创建一个解决方案名为 MyConsoleApp 的控制台应用程序
abp new MyConsoleApp -t console

# 新建项目

新建项目使用 abp new 命令，如：

# 新建一个解决方案为 Acme.MyConsoleApp 的控制台项目
abp new Acme.MyConsoleApp -t console
# 会创建数据库使用ef 界面使用mvc 的app 应用程序并且会创建react-native 移动应用程序
abp new <ProjectName>

常用命令组合，例如:

abp new <ProjectName> 
-t [app|console|module] 
-u [mvc|angular|blazor|none]
-m [none|react-native]
-d [ef|mongodb]
# 创建分层解决方案,Web和Http Api层在物理上是分开的.如果未指定会创建一个分层的解决方案,此解决方案没有那么复杂,适合大多数场景.
--tiered
# 将Identity Server应用程序与API host应用程序分开. 如果未指定,则服务器端将只有一个端点.
--separate-identity-server

# 更新项目

abp update [options]
  • 如果你的文件夹中有.sln文件,运行命令会将解决方案中所有项目ABP相关的包更新到最新版本.
  • 如果你的文件夹中有.csproj文件,运行命令会将项目中所有ABP相关的包更新到最新版本.

Options

  • --npm: 仅更新NPM包
  • --nuget: 仅更新的NuGet包

注: 这里仅记录了常用的，详细的请参考官方文档
