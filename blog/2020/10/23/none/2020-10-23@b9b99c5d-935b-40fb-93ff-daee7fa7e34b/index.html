Entity/Model/Dto等的区别
2020-10-23

Entity

数据表对应到实体类的映射，即：用代码描述一个数据表的结构

Model

Model 是 MVC 中的一个概念，可能与 Entity 不会一一对应，因为他只在 MVC 这个模型当中用于数据展示时用得到，他有可能是一个优化后的精简后的即对原始数据进行处理过的数据对象。

Dto

Dto 全称 (Data Transfer Object) 数据传输对象

Dto 对象与 Model 有些相似，但 Model 是 MVC 模式中的一个概念与 Dto 是有一定区别的。

Dto 多用于程序中数据的传输，比如程序从数据库中读取的数据映射为 Entity 对象，然后数据的流向需要向客户端传输，但用户可能不需要数据库中的某些数据，这时候 Dto 就可以对原始数据进行处理然后传往用户端。同样的客户端向程序传入数据时仅仅是用户方便处理的数据，但并不适合用于最后的数据存储，这时候将 Dto 转为 Entity 再保存到数据库当中。

Bean

对于Bean而言，只要是Java的类的就可以称为一个Bean。

更用在Spring上，被Spring管理的对象就可以将其称作为Bean。

它不仅仅可以包括对象的属性以及get,set方法，还可以有具体的业务逻辑。

POJO

除了属性和get、set方法外不包含具体的业务逻辑方法。

这个和 Model 很相像，和 Entity 区别在于没有和数据表中字段一一对应。

EJB

即EnterpriseBean，也就是Enterprise JavaBean（EJB）被称为Java企业Bean，是java的核心代码。

分别是会话Bean（Session Bean）、实体Bean（Entity Bean）、和消息驱动Bean（MessageDriven Bean）。
