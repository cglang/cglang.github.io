应用程序剖析

Main 方法

一个 ASP.NET Core 应用其实就是一个在其 Main 方法中创建一个 web 服务器的简单控制台应用程序

Main 调用遵循 Builder 模式的 WebHostBuilder，用于创建一个 web 应用程序宿主。

  • Host
    • CreateDefaultBuilder
      • 将一些环境变量以及配置文件加载到程序 IConfiguration。
      • 配置 ILoggerFactory 用于将日志等输出到控制台。
    • ConfigureWebHostDefaults
      • 启用 web 服务器 Kestrel，将 Startup 配置加载到程序等操作。

Startup 类

Startup 类是用来定义请求处理管道和配置应用需要的服务。Startup 类必须是public 并且包含下面两个方法

  • ConfigureServices 定义你的应用所使用的服务
  • Configure 定义你的请求管道中的 中间件(middleware)

更多参考

Services(服务)

服务的作用是用于在程序中通用调用的组件。服务通过依赖注入的方式获取并使用。ASP.NET Core 内置了一个简单的控制反转(IoC)容器，它默认支持构造器注入,并且他可以方便的更换为自己想要使用的容器(比如 Autofac)

Middleware(管道中间件)

ASP.NET Core 中间件为一个 HttpContext 执行异步逻辑，然后按管道顺序调用下一个中间件或者直接终止请求。

服务器(Servers)

ASP.NET Core 并不直接监听请求，他依赖于一个 名为 Kestrel 的 HTTP Server 来处理请求。

更多参考

内容根目录(Content root)

内容根目录是应用程序所用到的所有内容的根路径，例如它的 views 和 web 内容。内容根目录默认与宿主应用的可执行程序的应用根目录相同；一个替代的地址可以通过 WebHostBuilder 来设置。

Web根目录(Web root)

Web根目录(Web root)是项目中所有公共、静态资源(如 css、js、图片等)的目录。静态文件中间件将默认只发布 Web 根目录和其子目录中的文件。 Web 根目录默认为 /wwwroot，也可以通过 WebHostBuilder 来指定另外一个地址。

配置(Configuration)

ASP.NET Core 使用了一个新的配置模型用于处理简单的键值对。内置的配置提供者支持多种不同的文件格式如(XML，JSON， INI)和用于支持基于环境的配置环境变量。在 Main 方法调用 CreateDefaultBuilder 时默认会将 appsetting.json 加载到应用程序。

环境(Environments)

ASP.Net Core 从根据运行时环境的系统环境变量配置应用行为。
