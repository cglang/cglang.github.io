EF Core (二) EF 中的一些概念

DbContext

DbContext是实体类和数据库之间的桥梁，DbContext主要负责与数据交互，抛出抽象化用面向对象的方式我们可以把他当做一个数据库对象。

  1. DbContext 包含所有的实体映射到数据库表的实体集(DbSet<TEntity>)
  2. DbContext 将 LINQ-to-Entities 查询转换为SQL查询并将其发送到数据库。
  3. 更改跟踪: 它跟踪每个实体从数据库中查询出来后发生的修改变化。
  4. 持久化数据: 它也基于实体状态执行插入、更新和删除操作到数据库中。

通俗的讲就是我要对数据库进行的任何操作都要通过 DbContext 来进行

DbSet<TEntity>

DbSet 是从数据库中查询到的指定类型的所有实体的集合，通俗的讲就是他可以代表数据库中某张表的所有记录。

实体跟踪

在EF中框架会对你查询到的实体进行跟踪，以确保知道你对数据进行了什么样的修改，然后在保存修改时将修改对应到数据库中，查询到的实体共有五中状态分别是

  • detached：实体不在上下文的追踪范围内，如刚new的实体。
  • unchanged：未改变，如刚从数据库读取出来的实体。
  • added：添加状态，一般执行Add/AddRange时标记为added。
  • deleted：删除状态，一般执行Remove/RemoveRange时标记为deleted。
  • modified：修改状态，改变了实体的属性会处于这个状态。

状态的转变

  • added 状态的实体不能转为 deleted 和 modified 状态
  • deleted 状态的实体不能转为 added 状态
  • modified 状态的实体可转为 deleted 状态，不能转换为 added 状态。

一般不会手动改变实体的状态，除非有特殊需求。
