[ABP 系列] 创建第一个 MVC Web 应用程序 (一) 创建新的解决方案

发表于 2020-11-04 更新于 2020-11-05 阅读次数：
使用 ABP 创建一个 Web 应用程序

本文仅仅是实现 并没有讲解任何的代码部分 详细请参考官方文档

创建一个新的项目

新建一个文件夹，在文件夹内创建新的项目。

创建 Book 实体

在解决方案的领域层(.Domain`项目)中定义你的实体.

在Acme.BookStore.Domain项目中创建一个 Books 文件夹并在其中添加一个名为 Book 的类

添加 BookType 枚举

在 Acme.BookStore.Domain.Shared 项目创建新的文件夹 Books 并新建枚举类BookType

将 Book 实体添加到 DbContext 中

EF Core需要你将实体和 DbContext 建立关联.在Acme.BookStore.EntityFrameworkCore项目的BookStoreDbContext类中添加DbSet属性.

将 Book 实体映射到数据库表

在 Acme.BookStore.EntityFrameworkCore 项目中打开 BookStoreDbContextModelCreatingExtensions.cs 文件,添加 Book 实体的映射代码.

添加数据迁移

启动模板使用EF Core Code First Migrations创建和维护数据库架构.

打开菜单工具 > NuGet包管理器下的程序包管理控制台 (PMC).

注: 默认项目选择 Acme.BookStore.EntityFrameworkCore.DbMigrations

更新数据库

运行 Acme.BookStore.DbMigrator 应用程序来更新数据库。项目默认使用本地 SQL Server 数据库，如需使用其他数据库可自行更改数据库连接字符串。

.DbMigrator 是一个控制台使用程序,可以在开发和生产环境迁移数据库架构和初始化种子数据。

在.DbMigrator 项目中也有 appsettings.json 配置文件，里面也有数据库连接字符串这样的配置，使用的时候要注意是否和 .Web 项目的配置是否相同。

设置Acme.BookStore.DbMigrator 为启动项目，运行启动即可。

运行完之后你可以使用数据库管理工具查看数据库，已经为你建好一个名为 BookStore 的数据库。

创建应用程序

创建 BookDto

在 Acme.BookStore.Application.Contracts 项目中新建一个 Books 的文件夹，并创建一个名为 BookDto 的DTO类

创建 CreateUpdateBookDto

添加 实体与Dto 之间的映射

在Acme.BookStore.Application项目的BookStoreApplicationAutoMapperProfile类中使用AutoMapper库定义映射

创建 IBookAppService

在Acme.BookStore.Application.Contracts项目中定义一个名为IBookAppService的接口

创建 BookAppService

在Acme.BookStore.Application项目中创建名为 BookAppService 的 IBookAppService 实现

  • BookAppService继承了CrudAppService<...>.它实现了 ICrudAppService 定义的CRUD方法。

自动生成 API Controllers

ABP会自动按将你的应用程序服务配置为MVC API控制器，是你可以使用浏览器或第三方客户端通过AJAX调用它们.

Swagger UI

运行应用程序并在浏览器中输入https://localhost:XXXX/swagger/(用你自己的端口替换XXXX)作为URL.你会看到一些REST风格的接口。
